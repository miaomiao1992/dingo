Final standard: PascalCase (TitleCase) for all enum variants (Ok, Err, Some, None, Leaf, Node). Matches Rust/Kotlin/Haskell precedents, optimal for pattern matching readability, fully Go-compatible, no changes to current impl needed. Pros: Clear constructors, namespace-safe within enum; Cons of alternatives (camelCase/UPPER): Poor readability/var conflicts. Files compliant; add preprocessor validator.
